**Live site:** see the [GitHub Pages deployment](../../) for the rendered version.

## Repo layout

```
├── content/              Markdown source (31 files)
├── planning/             Locked decisions, preferences, agent style guide
├── draft.md              Original planning draft (kept as reference)
├── build.py              Markdown → HTML site generator
├── site/                 (Generated by build.py — not committed)
└── .github/workflows/    Auto-build on push, deploy to GitHub Pages
```

## Editing the trip

Every `.md` file under `content/` is rendered as an HTML page by `build.py`. To edit:

1. Make changes to the markdown
2. Push to `main`
3. GitHub Actions builds the site and deploys to Pages within ~2 minutes

To preview locally:

```bash
pip install markdown
python3 build.py
open site/index.html
```
